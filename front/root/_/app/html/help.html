Это – система учёта задач для небольшого (ориентировочно: до 50 сотрудников) творческого коллектива.

  • Не хранилище документации (сюда нельзя загружать файлы, кроме иллюстраций к репликам).
  • Не реализация регламентированных многошаговых процессов (здесь нет и не будет настройки ролей, групп, статусов, переходов и т. п.)
  • Не средство учёта рабочего времени (не стоит искать здесь отчёты по трудодням в разрезе проектов).
    • вообще проектов изначально не было и не предполагалось, но в определённый момент они всё же были добавлены.

А всего лишь база, хранящая данные о том:

  • кто кого когда о чём просил(а);
  • что было сказано в ответ и далее по ходу диалогов;
  • какие темы ещё актуальны, а какие были закрыты: когда именно и с какими словами.

Процесс

Диаграмма справа от этого абзаца иллюстрирует процесс, реализованный в системе. Он сводится к тому, что:

  • автор ставит задачу себе и какое-то время добавляет туда комментарии;
  • в некоторый момент он выбирает адресата и направляет задачу ему (при этом исполнитель получает e-mail со всеми репликами, написанными к этому моменту);
  • далее автор и адресат переписываются (добавляют комментарии) в рамках задачи, по обстоятельствам:
    • если необходимо, чтобы задача после комментария оказалась или осталась назначенной на корреспондента, кнопкой "мяч не на моей стороне"
    • в противном случае – кнопкой "пишу себе"
  • когда автор считает тему исчерпанной, он закрывает задачу кнопкой

Автор может выбрать адресата для задачи лишь единожды. При необходимости легко закрыть одну задачу, создать аналогичную, и направить её кому-то иному.

В любой момент времени каждая задача:

  • либо закрыта (статус "Архив");
  • либо в работе у одного их двух сотрудников:
    • то ли у адресата (статус "Назначено");
    • то ли у автора:
      • если адресат ещё не выбран – статус "Проект";
      • иначе – статус "Возвращено".

Адресат может сколько угодно переводить задачу автору с уточняющими вопросами, сообщениями о готовности, доводами в пользу отказа от затеи в принципе и т. п., но закрыть её имеет право лишь автор.

Адресат в рамках задачи переписывается только с её автором. Если необходимо участие коллег, можно ставить дополнительные задачи – но у каждой будет своя история, свой (также единственный) адресат и свой статус в каждый момент времени.

В течение всего процесса автор может и не выбрать адресата: такие задачи из проекта попадают сразу в архив. Это могут быть как нереализованные наброски, так и реальные задачи, реализованные автором самостоятельно.

В заключение отметим, что, по сравнению с аналогами, данная система не поддерживает следующие стандартные функции приёмы корпоративного саботажа:

  • перевод задачи на соседнее подразделение (отделов и рабочих групп просто нет);
  • перевод задачи на другого исполнителя (адресат определяется лишь однажды и имеет дело лично с автором);
  • обнуление ссылки на исполнителя (пока задача открыта, исполнитель определён однозначно);
  • добавление соисполнителей (такой объект отсутствует);
  • перевод задачи в экзотический статус, упущенный в отчётах.

Выбор корреспондетов

Чтобы получить возможность назначать задачи, пользователь должен предварительно выбрать своих адресатов из общего списка зарегистрированных в системе. Для этого ему следует открыть popup-окно третьей снизу слева кнопкой главного меню.

После сохранения данных кнопки с инициалами выбранных коллег будут показываться в правом нижнем углу страницы каждой задачи, для которой можно назначить исполнителя.

Реестр задач

Это основной экран системы. Естественная цель любого пользователя – минимизировать число строк, отображаемых там по умолчанию: то есть число задач, ждущих его действий.

Отображаемые столбцы имеют следующий смысл:

Дата
Дата/время создания задачи: когда автор впервые сформулировал её сам себе. По умолчанию сортировка идёт по этому полю, наверху – самые старые задачи.
Проект
Проект.
Тема
Тема задачи, указанная автором при её создании. Она не может меняться по ходу процесса. В первой строке имеется текстовый фильтр: он работает именно по полю "Тема".
Автор
Пользователь системы, создавший задачу. Click в первой строке открывает popup-окно со списком, где можно указать произвольное множество пользователей. Текущий пользователь отображается как "(я)", остальные – по имени и фамилии.

А если нет, но требуется выбрать единственного автора, указанного в конкретной ячейке – можно воспользоваться контекстным меню.

Адресат
Пользователь системы, которому задача адресована. Пока автор не указал адресата (что возможно для каждой задачи лишь единожды), он является адресатом сам, а задача пребывает в статусе "Проект". После выбора адресат фиксируется и далее меняться не может, в том числе и при закрытии задачи. Для этого поля работает точно такой же множественный фильтр, как для автора.
На ком
В отличие от (постоянного) адресата, здесь указан тот пользователь, от которого ожидаются какие-либо действия по задаче в текущий момент: то есть для возвращённых задач это автор, а для архивных – никто. По умолчанию для данного поля установлен фильтр по текущему пользователю системы, то есть в целом таблица содержит его актуальный список дел. Этот фильтр можно переключать так же, как для двух полей, указанных выше.
Статус
Вычислимое поле, значение которого определяется соответствием между автором, адресатом и текущим исполнителем. Правила описаны выше, и уже не раз. Доступна фильтрация по произвольному подмножеству из 4 возможных значений.
Последняя реплика / от
Пара полей, показывающих содержимое не задачи в целом, а последнего элемента связанной с ней переписки. По полю "от", то есть дате последней реплики, таблицу можно сортировать.

На верхней панели справа видны две кнопки. Они сбрасывают набор фильтров, каждая к своему предустановленному значению:

Что на мне?
"На ком" – текущий пользователь. Отображается список задач, требующих реальных действий от того, кто смотрит на экран в данный момент. Такой фильтр устанавливается по умолчанию при заходе в раздел.
Чего я жду?
"Автор" – текущий пользователь, статус – "Назначено". Отображается список задач, инициированных текущим пользователем, адресованных им кому-либо иному и в настоящий момент не возвращённых ни с результатом, ни с вопросами, ни с возражениями. Может быть полезно для руководящего работника.

Текствое поле на той же панели похоже на поле фильтра в столбце "Тема", однако оно шире: это поле именно расширенного поиска. Введённые туда значения ищутся как подстроки не только в заголовках, но во всём тексте переписки по задачам.

И, наконец, последнее. Для того, чтобы добавить новую задачу – нажмите кнопку "Добавить".
